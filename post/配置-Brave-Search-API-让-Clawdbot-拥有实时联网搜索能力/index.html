Clawdbot 配置 Brave Search API 详细步骤

方法一: 交互式配置(推荐)

这是最简单的配置方式,Clawdbot 会引导你完成所有设置。

bash
clawdbot configure --section web

执行后,按照提示操作:

  1. 选择启用 web_search 功能

  2. 粘贴你的 Brave Search API Key

  3. 确认配置完成

方法二: 配置文件方式

如果你熟悉命令行,可以直接编辑 Clawdbot 配置文件。

配置文件位置:

  • macOS/Linux: ~/.config/clawdbot/config.yaml

  • Windows: %APPDATA%\clawdbot\config.yaml

添加 Brave Search 配置:

yaml
# Clawdbot 配置文件
web_search:
  enabled: true
  provider: brave
  api_key: "sk-brave-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"

  # 可选: 搜索参数
  options:
    country: "US"        # 搜索结果地区
    search_lang: "en"    # 搜索语言
    count: 10            # 返回结果数量

方法三: 环境变量方式

适合 Docker 部署或 CI/CD 环境。

bash
# 设置环境变量
export BRAVE_API_KEY="sk-brave-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"

clawdbot start

Docker Compose 示例:

version: '3.8'

      - BRAVE_API_KEY=sk-brave-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

Brave Search API 在 Clawdbot 中的高级配置

搜索参数详解

Clawdbot 支持自定义 Brave Search 的多项参数:

参数 类型 说明 示例值 country string 搜索结果地区偏好 US, CN, JP search_lang string 搜索查询语言 en, zh-hans, ja ui_lang string 界面语言 en-US, zh-CN count int 返回结果数量 1-20 offset int 结果偏移量 0-9 safesearch string 安全搜索级别 off, moderate, strict freshness string 时间过滤 pd(24h), pw(周), pm(月)

完整配置示例

yaml
web_search:
  enabled: true
  provider: brave
  api_key: "sk-brave-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"

  options:
    # 基础设置
    country: "US"
    search_lang: "en"
    count: 10

    # 内容过滤
    safesearch: "moderate"

    # 高级选项
    text_decorations: true    # 启用文本高亮
    extra_snippets: true      # 获取额外摘要

Goggles 自定义搜索规则

Brave Search 独有的 Goggles 功能允许你自定义搜索结果排序和过滤规则。

yaml
web_search:
  provider: brave
  api_key: "your-api-key"
  options:
    goggles_id: "your-goggles-id"  # 应用自定义 Goggles

常用 Goggles 场景:

  • 排除特定域名

  • 优先展示技术文档站点

  • 过滤低质量内容农场

web_search:

  provider: brave
  api_key: "sk-brave-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"

  options:

    search_lang: "en"
    count: 10

    # 内容过滤
    safesearch: "moderate"

    # 高级选项

Brave Search API Clawdbot 配置常见问题

Q1: 配置后搜索功能不生效怎么办?

检查以下几点:

  1. 确认 API Key 正确: 重新复制粘贴,避免多余空格

  2. 检查计划类型: 必须是「Data for Search」计划,不是「Data for AI」

  3. 验证网络连接: 确保服务器可以访问 api.search.brave.com

  4. 查看日志: 运行 clawdbot logs --module web_search 查看错误信息
